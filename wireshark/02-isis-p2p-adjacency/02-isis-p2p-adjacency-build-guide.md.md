# Infrastructure Build Guide: Bare-Metal IS-IS L2 P2P Adjacency

> A visual and technical guide documenting the engineering, server provisioning, and validation of a pristine IS-IS routing protocol database exchange.

---

## Overview

This guide provides a comprehensive technical narrative of the IS-IS capture process. It follows the end-to-end engineering workflow: provisioning the bare-metal EVE-NG infrastructure, resolving Cisco IOL binary licensing constraints, applying point-to-point protocol configurations, and executing a pre-filtered packet capture pipeline to isolate the raw sequence.

---

## Phase 1: EVE-NG Server Setup & Licensing

Unlike standard QCOW2 images, Cisco IOL (IOS on Linux) binaries require a localized license file (`iourc`) mapped directly to the EVE-NG hostname. Without this, nodes will instantly drop into a continuous crash loop or throw a `Fatal Error: Connection Refused` upon booting.

### 1. Image Provisioning
SFTP into your EVE-NG server and upload your L3 Cisco IOL binary (e.g., `i86bi-linux-l3-adventerprisek9.bin`) to the following directory:
`/opt/unetlab/addons/iol/bin/`

### 2. License File Generation
To bypass the Cisco IOU validation checks, SSH into your EVE-NG CLI, navigate to the binary directory, download the community Python keygen script, and execute it to generate your host's specific license key structure:

```bash
cd /opt/unetlab/addons/iol/bin/
wget [https://gist.githubusercontent.com/paalfe/8edd82f780c650ae2b4a/raw/8267dccbe715edfc244e8bc1ff7af58d83a152e7/CiscoIOUKeygen.py](https://gist.githubusercontent.com/paalfe/8edd82f780c650ae2b4a/raw/8267dccbe715edfc244e8bc1ff7af58d83a152e7/CiscoIOUKeygen.py)
python3 CiscoIOUKeygen.py
````

The script will spit out an output formatted like this:


```
[license]
eveguesthostname = 123456789abcdef0;
```

Create a new file named `iourc` in that same folder, paste the exact text block generated by the script, and save it.

### 3. Server Permission Alignment

Because the files were created via root/external download, you must force the hypervisor to clean up and inherit the correct execution contexts. Run the native EVE-NG wrapper script to globally align permissions:


```
/opt/unetlab/wrappers/unl_wrapper -a fixpermissions
```

## Phase 2: Topology & Workflow Steps

### 1. Architecture and Topology

The lab environment features two Cisco IOL L3 routing nodes deployed inside the EVE-NG Web UI, linked directly via `Ethernet 0/0`. This direct connection eliminates intermediate layer 2 switching structures, ensuring a clean, isolated link-state environment.
![](../../screenshots/Screenshot%202026-06-22%20145659.png)

### 2. The Capture Pipeline (The Trap)

Before activating the routing processes, the packet capture pipeline was armed. We initiated the native EVE-NG Wireshark integration hook on the interconnecting link. A strict `isis` display filter was pre-applied to isolate the communication path from absolute zero, stripping out background infrastructure noise.
![](../../screenshots/Screenshot%202026-06-22%20152030.png)

### 3. Protocol Configuration & Adjacency Tuning

IS-IS processes were provisioned on both nodes using unique Network Entity Titles (NETs). To eliminate multi-access LAN behaviors—such as Designated Intermediate System (DIS) broadcast elections and heavy pseudo-node LSP generation—the physical interface links were explicitly locked down to `isis network point-to-point`.

**Router-2 (R2) Configuration Execution:**

### 4. Upstream Validation: Pristine IS-IS Database Sync

By establishing our capture state _before_ turning up the routing processes, the pipeline cleanly caught the complete protocol lifecycle. The final capture contains the exact initial `P2P HELLO` discovery handshake, followed by the immediate exchange of Complete Sequence Number PDUs (`CSNP`), Link State PDUs (`LSP`), and Partial Sequence Number PDUs (`PSNP`) to synchronize the link state databases. This is the zero-noise artifact merged upstream to the Wireshark Foundation.

## Phase 3: Copy-Paste Deployment Blocks

### EVE-NG Server Setup Commands


```
# Navigate to the IOL directory
cd /opt/unetlab/addons/iol/bin/

# Fetch the keygen bypass script
wget [https://gist.githubusercontent.com/paalfe/8edd82f780c650ae2b4a/raw/8267dccbe715edfc244e8bc1ff7af58d83a152e7/CiscoIOUKeygen.py](https://gist.githubusercontent.com/paalfe/8edd82f780c650ae2b4a/raw/8267dccbe715edfc244e8bc1ff7af58d83a152e7/CiscoIOUKeygen.py)

# Run the generator to copy the [license] block parameters
python3 CiscoIOUKeygen.py

# (After creating and pasting the keygen parameters into the 'iourc' file)
# Globally reset permissions so the hypervisor can boot the nodes
/opt/unetlab/wrappers/unl_wrapper -a fixpermissions
```

### Cisco Router-1 (R1) Configuration



```
conf t
hostname R1
router isis
 net 49.0001.0000.0000.0001.00
 is-type level-2-only
 metric-style wide
 log-adjacency-changes
 exit
interface Loopback0
 ip address 1.1.1.1 255.255.255.255
 ip router isis
 exit
interface Ethernet0/0
 ip address 10.0.0.1 255.255.255.252
 isis network point-to-point
 ip router isis
 no shutdown
 exit
end
```

### Cisco Router-2 (R2) Configuration



```
conf t
hostname R2
router isis
 net 49.0001.0000.0000.0002.00
 is-type level-2-only
 metric-style wide
 log-adjacency-changes
 exit
interface Loopback0
 ip address 2.2.2.2 255.255.255.255
 ip router isis
 exit
interface Ethernet0/0
 ip address 10.0.0.2 255.255.255.252
 isis network point-to-point
 ip router isis
 no shutdown
 exit
end
```

### Adjacency Reset (Trigger Packet Trap)



```
# Run this on R1 while Wireshark is listening to force an instantaneous clean database exchange sync
clear isis *
```
